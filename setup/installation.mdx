This guide walks you through every step needed to get both the React frontend and Java backend running on your local machine. See [requirements](/setup/requirements) for the full list of prerequisites.

## Clone the repository

<Steps>
  <Step title="Clone and enter the project">
    ```bash
    git clone https://github.com/Jhonmoreno000/PROYECTO-SENA-TIENDA-TEXTIL-
    cd "tienda digital de telas"
    ```

    The project root contains the React application source, the `backend-java/` directory, and the `database/` directory with the schema SQL.
  </Step>
</Steps>

## Frontend setup

<Steps>
  <Step title="Install npm dependencies">
    From the project root, install all frontend packages:

    ```bash
    npm install
    ```

    This command reads `package.json` and installs React 18, Vite 5, Tailwind CSS 3, React Router v6, Framer Motion, Recharts, and all other declared dependencies into `node_modules/`.
  </Step>

  <Step title="Review the Vite configuration">
    The Vite dev server is pre-configured in `vite.config.js`:

    ```javascript
    export default defineConfig({
      plugins: [react()],
      resolve: {
        alias: {
          '@': fileURLToPath(new URL('./src', import.meta.url)),
        },
      },
      server: {
        host: '0.0.0.0',
        port: 3001,
        strictPort: false,
        open: true,
        allowedHosts: true,
        proxy: {
          '/uploads': {
            target: 'http://localhost:8081',
            changeOrigin: true
          }
        }
      },
    })
    ```

    Key points:
    - The dev server listens on **port 3001**.
    - The `@` alias maps to `./src/` so you can import as `@/components/Header`.
    - The `/uploads` proxy forwards image requests to the Java backend on port 8081.

    <Note>
      You do not need to create a `.env` file. The frontend connects to the Java backend via the Vite proxy — no environment variables are required for local development.
    </Note>
  </Step>
</Steps>

## Backend Java setup

The backend lives in `backend-java/conexionPostgres/`. The source files are in `src/`, compiled output goes to `bin/`, and the required JARs are already in `lib/`.

### Directory structure

```
backend-java/conexionPostgres/
├── src/
│   ├── App.java              # Entry point — connects to DB, starts HTTP server
│   ├── conexion/
│   │   └── Conexion.java     # PostgreSQL JDBC connection (singleton)
│   ├── api/
│   │   ├── ApiServer.java    # HTTP server setup, route registration
│   │   ├── AuthHandler.java  # POST /api/login, POST /api/register
│   │   ├── UsersHandler.java # GET/PUT/DELETE /api/users
│   │   ├── OrdersHandler.java
│   │   ├── CouponsHandler.java
│   │   ├── ConfigHandler.java
│   │   └── SupportHandler.java
│   ├── dao/                  # Data access objects
│   └── models/               # Java model classes
├── lib/
│   ├── gson-2.10.1.jar
│   └── postgresql-42.7.3.jar
└── bin/                      # Compiled .class files (generated)
```

<Steps>
  <Step title="Compile the Java source">
    Compile all source files in `src/` to `bin/`, including the two JAR dependencies:

    <CodeGroup>

    ```bash Linux / macOS
    cd backend-java/conexionPostgres
    javac -cp "lib/gson-2.10.1.jar:lib/postgresql-42.7.3.jar" \
          -d bin \
          src/*.java src/**/*.java
    ```

    ```bash Windows (Command Prompt)
    cd backend-java\conexionPostgres
    javac -cp "lib/gson-2.10.1.jar;lib/postgresql-42.7.3.jar" ^
          -d bin ^
          src\*.java src\api\*.java src\conexion\*.java src\dao\*.java src\models\*.java
    ```

    </CodeGroup>

    <Tip>
      If you use VS Code with the Java Extension Pack, the IDE compiles automatically to `bin/` whenever you save. You can skip the manual `javac` step in that case.
    </Tip>
  </Step>

  <Step title="Set up the database connection">
    The backend reads the database connection from environment variables with fallback defaults defined in `src/conexion/Conexion.java`:

    | Environment variable | Default value | Description |
    |---|---|---|
    | `DB_URL` | `jdbc:postgresql://localhost:5432/tienda_digital_textiles_db` | Full JDBC URL |
    | `DB_USER` | `postgres` | PostgreSQL username |
    | `DB_PASSWORD` | `Mp.1025889078` | PostgreSQL password |

    If your local PostgreSQL uses a different password or port, set the environment variables before running:

    <CodeGroup>

    ```bash Linux / macOS
    export DB_USER=postgres
    export DB_PASSWORD=your_password
    export DB_URL=jdbc:postgresql://localhost:5432/tienda_digital_textiles_db
    ```

    ```bash Windows (Command Prompt)
    set DB_USER=postgres
    set DB_PASSWORD=your_password
    set DB_URL=jdbc:postgresql://localhost:5432/tienda_digital_textiles_db
    ```

    </CodeGroup>
  </Step>

  <Step title="Run the Java backend">
    Start the backend server from the `conexionPostgres` directory:

    <CodeGroup>

    ```bash Linux / macOS
    java -cp "bin:lib/gson-2.10.1.jar:lib/postgresql-42.7.3.jar" App
    ```

    ```bash Windows (Command Prompt)
    java -cp "bin;lib/gson-2.10.1.jar;lib/postgresql-42.7.3.jar" App
    ```

    </CodeGroup>

    A successful startup prints:

    ```
    Iniciando aplicación Backend...
    ✅ Conexión a PostgreSQL establecida con éxito.
    🚀 Servidor API escuchando en el puerto 8081
    ```

    <Warning>
      If you see `❌ No se pudo conectar a la base de datos`, the backend exits without starting the HTTP server. Make sure PostgreSQL is running and the database `tienda_digital_textiles_db` exists. See [database setup](/setup/database).
    </Warning>
  </Step>
</Steps>

## Start the application

With the backend running, start the React dev server in a separate terminal:

```bash
npm run dev
```

Both processes need to stay running simultaneously. Use two terminal windows or a terminal multiplexer.

## Access URLs

| URL | Service |
|---|---|
| `http://localhost:3001` | React frontend (Vite dev server) |
| `http://localhost:8081/api/products` | Java REST API |
| `http://localhost:8081/uploads` | Static product images |

## Production build

To create an optimized production bundle:

```bash
npm run build
```

Vite outputs the static files to `dist/`. Preview the production build locally with:

```bash
npm run preview
```

<Note>
  The production preview server also runs on port 3001 by default. The Java backend must still be running separately — it is not bundled into the Vite output.
</Note>
