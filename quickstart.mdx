This guide takes you from a fresh clone to a fully running local environment with both the React frontend and Java backend. The whole process takes about ten minutes.

## Prerequisites

Before you start, make sure you have the following installed:

| Tool | Minimum version | Notes |
|---|---|---|
| Node.js | 18 | Needed to run Vite and install npm packages |
| npm | 9 | Comes bundled with Node.js |
| Java JDK | 11 | The backend server is compiled and run with the JDK |
| PostgreSQL | 14 | The backend connects on `localhost:5432` by default |

## Steps

<Steps>
  <Step title="Clone the repository">
    Clone the project and navigate into the frontend root.

    ```bash
    git clone https://github.com/Jhonmoreno000/PROYECTO-SENA-TIENDA-TEXTIL-
    cd "tienda digital de telas"
    ```
  </Step>

  <Step title="Install frontend dependencies">
    Install all React, Vite, and Tailwind packages declared in `package.json`.

    ```bash
    npm install
    ```

    This installs React 18, React Router v6, Framer Motion, React Hook Form, Recharts, and all dev dependencies including Vite 5 and Tailwind CSS 3.
  </Step>

  <Step title="Create the database">
    Before starting the backend, you need a PostgreSQL database. Connect to PostgreSQL as the `postgres` superuser and create the database:

    ```sql
    CREATE DATABASE tienda_digital_textiles_db;
    ```

    Then load the full schema and seed data:

    <CodeGroup>

    ```bash Linux / macOS
    psql -U postgres -d tienda_digital_textiles_db -f database/schema.sql
    ```

    ```bash Windows (Command Prompt)
    psql -U postgres -d tienda_digital_textiles_db -f database\schema.sql
    ```

    </CodeGroup>

    <Note>
      The schema script drops and recreates all tables, so you can re-run it safely to reset the database to a clean state with seed data.
    </Note>
  </Step>

  <Step title="Start the Java backend">
    Navigate to the backend directory and launch the server. The backend compiles to the `bin/` folder and requires the two JARs in `lib/`.

    <CodeGroup>

    ```bash Linux / macOS
    cd backend-java/conexionPostgres
    java -cp "bin:lib/gson-2.10.1.jar:lib/postgresql-42.7.3.jar" App
    ```

    ```bash Windows (Command Prompt)
    cd backend-java\conexionPostgres
    java -cp "bin;lib/gson-2.10.1.jar;lib/postgresql-42.7.3.jar" App
    ```

    </CodeGroup>

    You should see:

    ```
    Iniciando aplicación Backend...
    ✅ Conexión a PostgreSQL establecida con éxito.
    🚀 Servidor API escuchando en el puerto 8081
    ```

    <Warning>
      If the backend prints `❌ No se pudo conectar a la base de datos`, check that PostgreSQL is running on port 5432 and that the database `tienda_digital_textiles_db` exists. See [database setup](/setup/database) for details.
    </Warning>
  </Step>

  <Step title="Start the React dev server">
    In a separate terminal, from the project root:

    ```bash
    npm run dev
    ```

    Vite starts on port 3001 by default (configured in `vite.config.js`). You should see:

    ```
    VITE v5.x.x  ready in Xms
    ➜  Local:   http://localhost:3001/
    ```
  </Step>

  <Step title="Log in with a demo account">
    Open [http://localhost:3001/login](http://localhost:3001/login) and sign in with one of the pre-seeded development accounts:

    | Role | Email | Password |
    |---|---|---|
    | Admin | `admin@ddtextil.com` | `admin123` |
    | Seller | `vendedor@ddtextil.com` | `vendedor123` |
    | Client | `cliente@ddtextil.com` | `cliente123` |

    <Warning>
      These are development-only credentials seeded into the local database. Do not use them in a production environment.
    </Warning>
  </Step>
</Steps>

## What to expect

Once both servers are running and you're logged in, you have access to all application routes:

| Route | Description |
|---|---|
| `/` | Homepage with hero, promotions carousel, and featured products |
| `/catalogo` | Full product catalog with category filters and price sorting |
| `/producto/:id` | Individual product detail page with image gallery |
| `/carrito` | Shopping cart with quantity controls and totals |
| `/checkout` | Checkout form with order summary and shipping calculation |
| `/login` | Authentication page |
| `/registro` | New account registration |

The Java backend API listens on port **8081** and the Vite dev server on port **3001**. The Vite proxy forwards `/uploads` requests to `http://localhost:8081` automatically.
