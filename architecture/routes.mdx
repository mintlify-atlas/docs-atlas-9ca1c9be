All routes are defined in `src/App.jsx`. The `ProtectedRoute` component guards every route that requires authentication or a specific role.

## `ProtectedRoute` usage pattern

```jsx
// src/components/ProtectedRoute.jsx
function ProtectedRoute({ children, roles }) {
    const { isAuthenticated, hasRole } = useAuth();
    const location = useLocation();

    if (!isAuthenticated) {
        return <Navigate to="/login" state={{ from: location }} replace />;
    }

    if (roles && !hasRole(roles)) {
        return <Navigate to="/" replace />;
    }

    return children;
}
```

- **No `roles` prop** — any authenticated user can access the route.
- **String `roles` prop** — user's role must match exactly (e.g. `roles="admin"`).
- **Array `roles` prop** — user's role must be in the array (e.g. `roles={['seller', 'admin']}`).
- Unauthenticated users are redirected to `/login` with the attempted path saved in `location.state.from`.
- Authenticated users without the required role are redirected to `/`.
